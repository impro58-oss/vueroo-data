## Issues Encountered & Fixes Applied

### 1. Gateway Auto-Start Zombie Process
**Problem:** Scheduled task spawning gateway at startup, causing conflicts when manual start attempted.
**Status:** Task deleted via `schtasks /Delete`
**Files Removed:**
- `scripts/auto-start-gateway.ps1`
- `scripts/openclaw-gateway-smart-start.ps1`
- `scripts/openclaw-gateway-start.ps1`
- `scripts/fix-gateway-task.ps1`

### 2. LLM Timeout / Model Not Loaded
**Root Cause:** Ollama unloads models after inactivity. OLLAMA_KEEP_ALIVE only works AFTER first load.
**Symptom:** Had to run `ollama run kimi-k2.5:cloud` manually before OpenClaw could connect.

**Fix Applied:**
- Set user-level env var: `OLLAMA_KEEP_ALIVE=60m`
- Restarted Ollama to pick up setting

**Pending Fix:** System-level warmup script needed
- Created `ollama-warmup.bat` in `.openclaw\`
- Scheduled task to run at login: `schtasks /Create /TN "Ollama Warmup"`

### 3. Model Configuration
**Updated in openclaw.json:**
- llama3 contextWindow: 8192 → 32000 (avoids "context too small" error)

## Commands for Reference

**Remove zombie startup:**
```cmd
del "%USERPROFILE%\.openclaw\workspace\scripts\*start*.ps1" /f
del "%USERPROFILE%\.openclaw\workspace\scripts\*start*.bat" /f
schtasks /Delete /TN "OpenClaw Gateway" /F 2>nul
```

**Set Ollama keepalive (user):**
```cmd
setx OLLAMA_KEEP_ALIVE "60m"
```

**Set Ollama keepalive (system):**
```cmd
setx OLLAMA_KEEP_ALIVE "60m" /M
```

**Create warmup task:**
```cmd
schtasks /Create /TN "Ollama Warmup" /TR "%USERPROFILE%\.openclaw\ollama-warmup.bat" /SC ONLOGON /RL LIMITED /F
```

## Key Learning
OLLAMA_KEEP_ALIVE does NOT preload models. It only keeps them loaded AFTER first use. Need warmup script at startup.
