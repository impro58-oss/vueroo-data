# Auto-Start Experiment Log - 2026-03-17

## Summary
Attempted to configure OpenClaw gateway to auto-start on Windows boot with Chrome auto-launch. Multiple approaches tested, all failed due to Windows UAC/elevation requirements.

## What Was Attempted

### Attempt 1: Startup Folder (.cmd file)
- Created `OpenClaw Gateway.cmd` in Startup folder
- **Failed:** UAC prompt required admin approval
- **Why:** Batch files trigger UAC when running node.exe

### Attempt 2: Scheduled Task (Official)
- Used `openclaw gateway install` to create task
- **Failed:** Still required elevation/password
- **Why:** Default task runs with user privileges but Windows security blocks

### Attempt 3: VBS Wrapper (Hidden window)
- Created VBS script to launch gateway silently
- **Failed:** Gateway started but Chrome didn't open visible window
- **Why:** Chrome already running, new window not forced properly

### Attempt 4: Modified VBS with --new-window
- Added Chrome flags to force new window
- **Failed:** Still had UAC issues, timing problems
- **Why:** Underlying elevation requirement not resolved

## Root Cause

**Windows UAC Security:** Any background service that binds to ports or runs persistently requires either:
1. Admin elevation (UAC prompt)
2. Proper service registration with SYSTEM privileges
3. User logged in with specific permissions

**OpenClaw Gateway:** Requires port binding (18789) which Windows treats as privileged operation when auto-starting.

## What Works

**Manual Start (100% reliable):**
```powershell
openclaw gateway start
# Then open Chrome to http://127.0.0.1:18789/
```

## System Changes Made

### Kept:
- Sleep disabled (PC stays awake)
- Hibernate disabled
- Windows Update auto-restart disabled
- Metered connection set (blocks background updates)

### Removed:
- All startup files from `%APPDATA%\Microsoft\Windows\Start Menu\Programs\Startup\`
- All scheduled tasks for OpenClaw
- All auto-launch scripts

- `scripts/setup-openclaw-autostart.ps1` - Task scheduler setup

## Key Learnings

1. **UAC is the blocker:** No automation works without either admin rights or user present to click "Yes"
2. **Scheduled tasks need proper privilege configuration:** `/rl LIMITED` helps but still has edge cases
3. **VBS can hide windows but not bypass UAC:** Elevation happens at process creation
4. **Timing matters:** Gateway needs 10-15 seconds to initialize before Chrome connects
5. **Chrome already running = no new window:** Must use `--new-window` flag

## For Remote Access

**Current solution:** Remote Desktop into machine, manually run `openclaw gateway start`

**Future options:**
- Run OpenClaw on VPS (already configured, no UAC issues)
- Configure Windows to auto-login (security risk)
- Use service account with proper permissions (complex)

## Recommendation

**Accept manual start.** The automation attempts consumed significant time and introduced system instability. Manual start is reliable, simple, and works every time.
