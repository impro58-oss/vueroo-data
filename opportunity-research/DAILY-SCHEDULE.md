# Daily Opportunity Research Schedule

## Schedule Configuration

**Frequency:** Daily  
**Preferred Time:** 09:00 UTC (morning)  
**Duration:** 30-60 minutes per session  
**Mode:** Prompted (awaiting user confirmation before starting)

## Daily Flow

### 09:00 — Session Trigger
- System creates session marker
- Notification sent (if configured)
- User reviews yesterday's offshoots
- User confirms or reschedules

### 09:05 — Research Execution
- Step 1: Review prior context (5 min)
- Step 2: Map signal strength (5 min)
- Step 3: Direct signal research (10 min)
- Step 4: Indirect signal research (10 min)
- Step 5: Cross-pollination (5 min)
- Step 6: Generate concepts (10 min)
- Step 7: Score and rank (5 min)
- Step 8: Plan tomorrow's offshoots (5 min)

### 09:45 — Documentation
- Update daily-research-log.md
- Score concepts
- Commit to opportunity-map.md
- Update branch tracker

### 09:55 — Sync Push
- `/sync push` with session summary
- Update STATE.md with new opportunities

## Weekly Review

**Every Sunday:**
- Review top 3 opportunity clusters
- Assess signal strength trends
- Kill list: dismiss weak concepts
- Promote strong concepts to validation

## Monthly Deep Dive

**First Monday of each month:**
- Select 1 concept for validation testing
- Build MVP or landing page
- Run validation experiment
- Document learnings
